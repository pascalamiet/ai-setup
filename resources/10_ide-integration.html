Neovim / Vim

If you're a Vim/Neovim user, you already live in the terminal. Claude Code is a perfect fit ‚Äî run it in a split terminal pane right next to your editor buffer.

The simplest setup: tmux split panes

# In tmux: split horizontally, run Claude on the right
# Default prefix is Ctrl+B
Ctrl+B  %         # vertical split
Ctrl+B  "         # horizontal split
                  # then start claude in the new pane
claude

Neovim terminal split (no tmux needed)

" In Neovim ‚Äî open Claude in a vertical terminal split
:vsplit | terminal claude

" Or bind it to a leader key in init.lua:
vim.keymap.set('n', 'ai', function()
  vim.cmd('vsplit | terminal claude')
end, { desc = 'Open Claude Code' })

Recommended .tmux.conf additions

# .tmux.conf ‚Äî quick Claude pane
# Prefix + C opens Claude in a right split
bind-key C split-window -h "claude"

# Larger right pane for Claude (60/40 split)
bind-key C split-window -h -p 40 "claude"

Plugin options for Neovim

Several community plugins integrate Claude Code more deeply into Neovim:

https://github.com/greggh/claude-code.nvim
https://github.com/coder/claudecode.nvim

Install a Neovim plugin (lazy.nvim)

-- In your lazy.nvim plugins config
{
  "greggh/claude-code.nvim",
  dependencies = { "nvim-lua/plenary.nvim" },
  config = function()
    require("claude-code").setup({
      window = {
        position = "vertical",  -- "vertical" | "horizontal" | "float"
        size = 0.4,               -- 40% of screen
      }
    })
    -- Toggle Claude with <leader>ac
    vim.keymap.set("n", "<leader>ac", "<cmd>ClaudeCode<cr>")
  end,
}

üíª Terminal Setup Inside Your IDE

Whatever IDE you use, you'll get the best experience by running Claude Code in an integrated terminal panel rather than switching to a separate terminal app. Here's how to configure the integrated terminal for Claude Code.

Use your preferred shell

# VS Code settings.json ‚Äî set default shell
{
  "terminal.integrated.defaultProfile.osx": "zsh"
}
// VS Code settings.json ‚Äî set default shell
{
  "terminal.integrated.defaultProfile.windows": "PowerShell"
  // Or use Git Bash: "Git Bash"
  // Or use WSL: "Ubuntu (WSL)"
}

Open Claude in a persistent IDE terminal

# Run Claude in interactive mode in your IDE terminal
claude

# Or open it in a specific project directory
claude --project /path/to/project

Recommended terminal font

Claude Code's output uses Unicode characters for formatting. A Nerd Font renders them properly:

https://www.nerdfonts.com/font-downloads 
// VS Code ‚Äî set terminal font to a Nerd Font
{
  "terminal.integrated.fontFamily": "JetBrainsMono Nerd Font"
}

‚ö° Side-by-Side Workflow Tips

1. Let Claude handle the boilerplate; you handle the architecture

Open your code in the editor, open Claude in the terminal. You read the code and make decisions about structure. You tell Claude to implement the pieces. Review its changes as diffs in the editor. Merge what's good.

2. Reference files by path in your Claude prompts

# Claude Code reads files in your project automatically
# But you can reference specific files explicitly
Look at src/lib/auth.ts and tell me how the session validation works.

# Or ask it to look at what's currently open
Read the file I was just working on (UserCard.tsx) and refactor the avatar logic.

3. Use the editor's git diff view to review Claude's changes

After Claude edits files, use your IDE's git panel to see exactly what changed. VS Code's Source Control view (‚åòCtrl+Shift+G) shows line-by-line diffs. JetBrains' Local Changes tab does the same. This is much more readable than trying to follow Claude's output in the terminal.

4. Auto-save is your friend

Enable auto-save in your IDE so Claude always reads the latest version of files you're editing. Without it, Claude might read a stale version from disk while you have unsaved changes in the editor.

5. Keep Claude's working directory correct

# Start Claude from the project root ‚Äî not a subdirectory
# Claude uses the current directory to understand your project
cd ~/projects/my-app
claude

6. Use multiple terminal tabs

In VS Code and JetBrains, you can have multiple terminal tabs. Keep one running Claude, one for running tests/builds, and one for git. Switching between them is much faster than switching windows.

üé≠ The ideal setup: Claude in the right terminal pane, your code in the editor, and a test runner in the bottom pane. Three panes of information, zero context-switching, and the ambient anxiety of watching an AI edit your codebase in real time. Exciting.

‚å®Ô∏è Global Keybindings

Claude Code supports custom keybindings that work inside its interactive session. These are configured separately from your IDE's keybindings.

Keybindings config location

~/.claude/keybindings.json
%USERPROFILE%\.claude\keybindings.json

Example custom keybindings

[
  {
    "key": "ctrl+shift+e",
    "command": "workbench.action.terminal.focus"
  },
  {
    "key": "escape escape",    // chord: press Esc twice to interrupt Claude
    "command": "claude.interrupt"
  }
]
