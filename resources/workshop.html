Getting Oriented

Learn the Claude Code interface before diving into tasks.

Beginner ~5 min

What you'll learn

  • How to start and exit Claude Code
  • Key slash commands for navigation
  • How the agent reads your current directory
1

Launch Claude Code in your home directory (or anywhere):

claude
Claude will greet you and show the current directory. It automatically reads any CLAUDE.md file it finds, giving it project context.
2

Explore the built-in slash commands:

# Inside Claude Code — type these at the prompt
/help
/status
/model
3

Ask Claude something simple to verify it works:

Type to Claude What files are in my current directory? Give me a brief summary.
4

Exit when done. There are three ways:

/exit          # slash command
exit           # plain text
Ctrl+C         # keyboard shortcut (works on both OS)

  • How to save search results to a file

Create a data analysis script in your preferred language:

4

Spawn a Subagent

Delegate parallel workstreams to background AI workers.

Medium ~15 min

What you'll learn

  • What a subagent is and when to use one
  • How to craft prompts that trigger parallel agent delegation
  • How to collect and merge results from multiple agents
What is a subagent? When Claude Code faces a complex task, it can spin up background worker agents to handle subtasks in parallel — like hiring an assistant who hires their own assistants. You'll see this in the interface as Task tool calls. Each subagent gets its own context, runs independently, and reports back.
1

Ask Claude to do two research tasks at once using subagents:

Type to Claude I need you to do two independent research tasks simultaneously using subagents — don't do them sequentially.

Subagent 1: Search the web for the top 3 methodological approaches used in [your field] papers from 2024. Summarize each in 3 bullet points.

Subagent 2: Search for software tools or datasets commonly used by researchers in [your field]. List the top 5 with a one-line description each.

When both are done, combine the results into a file called notes/field-overview.md.
💡 Watch the interface — you should see Claude invoke the Task tool twice (almost simultaneously). Each subagent runs in parallel and reports back independently.
2

Use subagents for a deeper literature comparison:

Type to Claude Use two subagents to compare two schools of thought in [your field]:

Subagent 1: Research the arguments and key papers supporting [Approach A].

Subagent 2: Research the arguments and key papers supporting [Approach B].

Then write a structured comparison in literature/comparison.md with a markdown table showing key differences.
3

Now try a code + research combination — one agent codes, one researches:

Type to Claude Run two tasks in parallel:

Subagent 1: Create a Python script analysis/fetch_papers.py that uses the Semantic Scholar API to search for papers by keyword and saves results as JSON.

Subagent 2: Search the web for the Semantic Scholar API documentation and summarize the key endpoints and rate limits.

Once both finish, update the script using the API documentation the second agent found.
Expected outcome Two independent tasks complete faster than sequential execution would allow, then merge into a final deliverable.

5

Build a Custom Subagent

Define a persistent, specialized AI agent tailored to your research domain.

Medium ~15 min

What you'll learn

  • The difference between ad-hoc subagents and defined custom agents
  • How to write an agent definition file with a targeted system prompt
  • How to control which tools your agent has access to
  • How to invoke your custom agent by name
Ad-hoc vs. custom subagents: In Task 4 you asked Claude to spawn workers on the fly. A custom subagent is different — it's a reusable agent you define once with its own name, system prompt, and tool restrictions. Claude automatically knows when to use it based on its description, and you can invoke it explicitly by name.
1

Create the agents directory for this project:

# In your terminal (NOT inside Claude), while in ai-workshop/
mkdir -p .claude/agents
# In PowerShell (NOT inside Claude), while in ai-workshop\
New-Item -ItemType Directory -Force -Path ".claude\agents"
2

Create your custom literature reviewer agent. The file has two parts: a YAML frontmatter block that defines the agent's identity and tool access, and a markdown body that is its system prompt.

cat > .claude/agents/literature-reviewer.md << 'EOF'
---
name: literature-reviewer
description: Use this agent when the user asks to find, summarize, or
  critically analyze academic papers on a topic. Invoke automatically
  for any literature search, paper review, or research synthesis request.
tools:
  - WebSearch
  - WebFetch
  - Write
  - Read
---

You are a specialized academic literature reviewer for a PhD researcher.

Your responsibilities:
- Search for peer-reviewed papers using Google Scholar, Semantic Scholar,
  arXiv, and field-specific databases
- For each paper extract: title, authors, year, venue, key contribution,
  methodology, main findings, and limitations
- Identify recurring themes, contradictions, and open debates across sources
- Always distinguish between well-established results and preliminary findings
- Prioritize recency (last 3 years) unless foundational papers are requested
- Flag if you cannot access the full text of a paper

Output format (for each paper):
**Title** — Authors (Year) *Venue*
- Contribution: one sentence
- Method: one sentence
- Finding: one sentence
- Limitation: one sentence

End every response with a "Synthesis" section that groups papers thematically
and identifies the most important gap in the literature.
EOF
@'

💡 The description field is what Claude reads to decide when to automatically invoke this agent. Make it specific and action-oriented. The tools list limits what the agent can do — a web-only agent can't accidentally edit your files.
3

Start Claude and invoke your agent explicitly by name:

claude
Type to Claude Use the literature-reviewer agent to find 4 recent papers on [your specific research topic]. Save the results to literature/agent-review.md.
Expected outcome Claude delegates to your custom agent, which applies the structured output format and synthesis section you defined — consistently, every time.
4

Customize the agent for your field. Edit the system prompt directly:

Type to Claude Edit .claude/agents/literature-reviewer.md to specialize it further: add a line saying it should prioritize papers from these journals: [list 3 journals in your field]. Also add that it should always note whether a paper uses quantitative or qualitative methods.
💡 This is the key advantage over a one-off prompt: you refine the agent once and every future invocation benefits. It's the difference between a colleague who remembers your preferences and one who needs to be briefed every time.
5

Create a second agent — a data analyst with code execution access:

Type to Claude Create a second agent at .claude/agents/data-analyst.md. This agent specializes in exploratory data analysis for my field. It should have access to Bash, Read, Write, and Edit tools. Its system prompt should: always start by checking what data files exist, prefer pandas and matplotlib for Python, document every transformation it applies, and save all plots to output/figures/.

📂 Agent file anatomy

---  ← YAML frontmatter (required)
name: my-agent-name          ← used to invoke it by name
description: When to use...  ← Claude reads this to auto-route tasks
tools:                         ← restrict tool access (omit = all tools)
  - WebSearch
  - Write
---  ← end of frontmatter

Your system prompt goes here.   ← plain markdown, as long as you want
Define the agent's persona, rules, output format, constraints...

Write a Custom Skill

Create reusable slash commands for your most common workflows.

Medium ~15 min

What you'll learn

  • Where skills (slash commands) live and how they work
  • How to write a reusable prompt template
  • The difference between project-level and user-level skills
How skills work: A skill is just a .md file containing a prompt. Drop it in .claude/commands/ and it becomes available as /filename inside Claude Code. Skills can live at the project level (inside your project folder) or globally (~/.claude/commands/).
1

Create the skills directory for this project:

# In your terminal (NOT inside Claude), while in ai-workshop/
mkdir -p .claude/commands
# In PowerShell (NOT inside Claude), while in ai-workshop\
New-Item -ItemType Directory -Force -Path ".claude\commands"
2

Create your first skill — a research note generator:

# Create the skill file (edit with any text editor, or just use cat)
cat > .claude/commands/research-note.md << 'EOF'
Create a structured research note from our current conversation.

Format as:
# Research Note: [infer topic from conversation]

**Date**: [today's date]
**Status**: Draft

## Key Findings
- [bullet points of the most important insights]

## Open Questions
- [questions raised but not answered]

## Next Steps
- [concrete actions I should take]

## References
- [any papers, URLs, or sources mentioned]

Save the file to `notes/[YYYY-MM-DD]-[topic-slug].md`.
Confirm the path when done.
EOF
# Create the skill file in PowerShell
@'
Create a structured research note from our current conversation.

Format as:
# Research Note: [infer topic from conversation]

**Date**: [today's date]
**Status**: Draft

## Key Findings
- [bullet points of the most important insights]

## Open Questions
- [questions raised but not answered]

## Next Steps
- [concrete actions I should take]

## References
- [any papers, URLs, or sources mentioned]

Save the file to `notes/[YYYY-MM-DD]-[topic-slug].md`.
Confirm the path when done.
'@ | Set-Content .claude\commands\research-note.md
3

Test your skill — start Claude and invoke it:

claude
# Inside Claude — first have a quick conversation, then run the skill:
/research-note
Expected outcome Claude creates a formatted note in your notes/ folder — structured, dated, and ready to use.
4

Create a second skill — a quick paper summarizer:

Type to Claude (ask it to create the skill for you!) Create a skill file at .claude/commands/summarize-paper.md. The skill should ask me to paste or describe a paper, then output: (1) one-sentence summary, (2) key contribution, (3) methodology in 3 bullets, (4) limitations, (5) relevance to my research. Format it as a clean markdown block I can copy.
💡 Meta-moment: you just used Claude to write a skill for Claude. This is recursive and intentional.
5

Create a global skill (available across all projects):

# Global skills go in ~/.claude/commands/
mkdir -p ~/.claude/commands

# Create a "commit message" helper

EOF

Capstone: Build a Research Assistant

Apply everything you've learned in one end-to-end workflow.

Advanced ~25 min

What you'll build

A lightweight literature review assistant that: searches for papers, extracts key ideas, compares approaches, generates a structured review section, and saves everything in a usable format.

1

Set the stage. Update your CLAUDE.md to focus on a specific mini-review task:

Type to Claude Update CLAUDE.md to add a new section called "Current Task" that says: I am writing a mini literature review (500–700 words) on [specific topic in your field]. The goal is a survey of 4–6 papers published between 2022–2025. Target journal: [name a journal in your field or use "general academic audience"].
2

Search in parallel. Use subagents to gather papers from multiple angles:

Type to Claude Use three subagents simultaneously to search the web:

Subagent 1: Find 2 foundational/theoretical papers on [topic] from 2022–2024.
Subagent 2: Find 2 applied/empirical papers on [topic] from 2023–2025.
Subagent 3: Find any recent reviews or meta-analyses on [topic].

Save the full combined results to literature/capstone-sources.md in a structured format with title, authors, year, abstract excerpt, and a "why include" note.
3

Synthesize. Turn raw sources into structured analysis:

Type to Claude Based on literature/capstone-sources.md, create literature/capstone-analysis.md with: 1. A table comparing all papers by methodology, dataset, key finding, and limitation. 2. Three thematic groupings that emerge from these papers. 3. One key gap or open question that this body of work leaves unanswered.
4

Write the draft. Generate the actual literature review:

Type to Claude Write a 500–700 word literature review section based on the analysis in literature/capstone-analysis.md. Use an academic tone. Structure it as: opening paragraph (context), one paragraph per thematic group, closing paragraph (gap + research opportunity). Save to output/literature-review-draft.md.
Expected outcome A fully-formed literature review draft — not perfect, but a solid first draft you could actually build on for a paper.
5

Use your skill. Save a research note on the whole session:

# Inside Claude Code
/research-note
6

Reflect. Ask Claude to evaluate its own output:

Type to Claude Be critical: what are the three weakest points in the literature review you just wrote? What would a reviewer most likely flag? Suggest specific improvements.
💡 Teaching the agent to critique its own work is one of the highest-leverage skills you can develop. It's faster than you editing it blind.

Bonus A: Hooks — Automate on Events

Trigger shell commands automatically when Claude takes certain actions.

Bonus Advanced ~15 min
What are hooks? Hooks let you run shell commands in response to Claude's actions — for example, automatically logging every file Claude edits, playing a sound when a task finishes, or running tests after Claude writes code. They live in your settings file.
1

Open your Claude Code settings file in a text editor:

# Exit Claude first, then open settings
open ~/.claude/settings.json
# Or use any editor:
code ~/.claude/settings.json
nano ~/.claude/settings.json
# Exit Claude first, then open settings in Notepad
notepad $HOME\.claude\settings.json
# Or VS Code:
code $HOME\.claude\settings.json
2

Add a hook that logs every file Claude writes or edits:

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date '+%Y-%m-%d %H:%M:%S') EDITED: $CLAUDE_TOOL_INPUT_FILE_PATH\" >> ~/ai-workshop/agent-activity.log"
          }
        ]
      }
    ]
  }
}
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -Command \"Add-Content -Path \\\"$HOME\\\\ai-workshop\\\\agent-activity.log\\\" -Value \\\"$(Get-Date -f 'yyyy-MM-dd HH:mm:ss') EDITED: $env:CLAUDE_TOOL_INPUT_FILE_PATH\\\"\""
          }
        ]
      }
    ]
  }
}
⚠️ If your settings.json already has content, merge carefully — don't overwrite the whole file. Only add the "hooks" key.
3

Restart Claude and ask it to create a test file. Then check the log:

# After Claude creates a file, in a new terminal tab:
cat ~/ai-workshop/agent-activity.log
# After Claude creates a file, in a new terminal:
Get-Content $HOME\ai-workshop\agent-activity.log
Expected outcome Every file Claude touches gets logged with a timestamp. You have a full audit trail of agent activity.
4

Challenge: Add a second hook that runs your test suite after any Python file is modified.

Type to Claude (let it help you write the hook!) Help me write a PostToolUse hook that runs python -m pytest analysis/ automatically whenever Claude edits a .py file. It should print PASS or FAIL to the terminal.

Bonus B: Power-User CLAUDE.md

Turn your project memory into a full instruction set for a research co-pilot.

Bonus Medium ~15 min
A minimal CLAUDE.md gets you 20% of the benefit. A well-crafted one gets you 80%. This task shows you what a production-quality CLAUDE.md looks like for research.
1

Ask Claude to audit your current CLAUDE.md and suggest improvements:

Type to Claude Read my current CLAUDE.md and critique it. What's missing? What would make you significantly more useful in this project? Give me a list of concrete additions to consider.
2

Have Claude rewrite CLAUDE.md with advanced sections:

Type to Claude Rewrite my CLAUDE.md to include these sections:

1. Project Overview — what I'm doing and why
2. My Research Style — how I think, what I value (rigor, brevity, citations)
3. Agent Behavior Rules — always ask before deleting files; prefer edits over rewrites; cite sources when searching the web; flag when you're uncertain rather than guessing
4. Coding Standards — language preferences, docstring style, error handling approach
5. File Conventions — naming, folder structure, date format (ISO 8601)
6. Available Skills — list my custom skills with a one-line description each
7. Current Research Focus — the active question I'm working on
Expected outcome A CLAUDE.md that makes every future session start with Claude already knowing your full research context, preferences, and constraints.
3

Test it: exit, restart, and ask a tricky context question:

Type to Claude (after restarting) Without me telling you anything: what should you do if you're about to delete a file? What date format should you use for filenames? What's my preferred programming language?
💡 If Claude answers all three correctly without you saying a word, your CLAUDE.md is working. This is the difference between a generic assistant and your personal research co-pilot.
